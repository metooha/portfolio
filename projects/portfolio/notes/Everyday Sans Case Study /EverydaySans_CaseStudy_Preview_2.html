Everyday Sans UI

Brand. Performance.
Pick one — or used to.

Every new font weight came with a performance cost. So Brand held back. The result: visual hierarchy that never reached its potential, and a design system that had to make peace with compromise. The variable font ended that trade-off — permanently.

The compromise we stopped making

Our font system was built on static files — one file per weight. Brand couldn't introduce Medium, Semibold, or Black without Engineering absorbing the performance cost of each one. The result: years of visual hierarchy limited to two weights.

Brand

Needed weight variety

Price callouts, headings, body copy, and display text all need different weights to create clear hierarchy. With only Regular and Bold, every surface looked visually flat.

vs
Engineering

Couldn't absorb the cost

Adding the Medium weight file alone was blocked — it pushed page load times past the web vitals threshold. Each new file meant another HTTP request on every page load.

✓

How it was resolved: The variable font makes weight a continuous axis — not a collection of files. Engineering pays the performance cost once (one file, one request). Brand gets the full weight scale. The constraint that blocked both teams no longer exists.

The Journey

It wasn't a straight line.

Getting to a system where Brand doesn't have to compromise performance took 18 months, six versions, and three organizations learning to want the same thing.

Jun 2024
High hopes. Early friction.
Brand Foundry A11y

The foundry delivered a first Bold draft. The team's reaction was blunt: too heavy, too tight, and letters that looked too similar to each other. The A11y team flagged failures immediately. Not the start anyone wanted.

Foundry vs. A11y on what "readable" actually means — at every weight and every size.
Late 2024
Engineering hits a wall.
Engineering iOS Android

When Engineering tried to add the Medium weight mid-cycle, it broke the release — too heavy for web vitals. Then a harder blocker surfaced: iOS doesn't support variable fonts at all. The file came in at 70kb — more than twice the 30kb limit. Each platform had its own version of no.

The file that was supposed to replace everything couldn't run on half our devices. iOS needed static files. Web needed it under 30kb. Android needed its own workarounds.
Jan 2025
First real alignment.
Brand Engineering Foundry

The team landed on a shared character set — reducing from 541 glyphs to 179. Brand gave up some special characters. Engineering got a leaner file. The first time all three sides agreed on the same thing.

Brand, Engineering, and Foundry aligned on a shared glyph set for the first time.
Jul 2025
The great scale debate.
Brand Engineering Foundry

The foundry presented two paths: scale all text up ~8.5%, or scale down ~8%. Every layout, every component, every screen would shift either way. Brand wanted consistency with Bogle. Engineering didn't want to rework layouts. Two weeks of back-and-forth before the team chose to scale up — a harder short-term lift, but the right long-term call for font system consistency across Walmart.

Brand wanted visual parity with Bogle. Engineering didn't want to absorb layout rework. Two legitimate requirements, directly in conflict.
Aligned on scaling up — short-term rework accepted in exchange for long-term font system consistency.
Sep–Oct 2025
Two steps forward, one step back.
Engineering A11y Design

Testing on real Walmart product pages kept surfacing new problems: strikethrough landing across the middle bar of the number 4, making prices hard to read. Text off-center in buttons. Mono font inconsistent with the UI font. Each fix revealed another edge case. The team was in a loop — foundry delivers, Engineering finds an issue, Design confirms, foundry revises. Slow. Frustrating. But working.

Teams debated whether to keep iterating or ship and fix post-launch. Nobody wanted to go backwards — but nobody wanted to ship something broken.
Dec 2025
All parties sign off.
Brand Engineering A11y Foundry

Final file delivered. Variable font plus five static weights, every platform confirmed. Engineering validated file sizes. A11y signed off on legibility. Brand confirmed it felt right next to Bogle. After 18 months and six versions, four organizations agreed on the same file.

Brand, Engineering, A11y, and Commercial Type all signed off on the same final file.
